# Hackathon Win Strategy

**Event:** [Cursor Hands Off London Hackathon](https://cursor-hands-off-hackathon-06-2026.vercel.app/)  
**Product name:** Datter AI Hands-Off  
**Goal:** Top 5 live demo → podium

## Event thesis (what judges want)

Build a **self-running business powered by AI agents** — set up workflows, **let go**, come back to see what worked.

**Not:** best ML paper or generic Streamlit table.

**Pitch reframe:** Datter is an **autonomous data-ingestion company** — agents audit a corpus, find the optimal cut on the quality–cost curve, leave an audit trail.

## Demo format (hard requirements)

| Time | Action |
|---|---|
| **21:30** | Code freeze — submit project + **Loom/YouTube ≤2 min** |
| **22:35** | Top 5 **live demo** (2 min each) |
| **22:55** | Awards |

Prepare **video + live run** backup.

## Judging criteria → Datter response

| Criterion | How we win |
|---|---|
| Technical execution | Local pipeline + eval loop works offline-first |
| Product thinking | Project-scoped corpus + clear ICP ([[Project Model]]) |
| Agent autonomy | Streaming agent log + auto-run / hands-off mode |
| UX clarity | Outcome-first: **understanding % + token savings**, not chunk tables first |
| Real-world applicability | [[Vertical Demo Corpus]] — gov/social/eng/science PDFs |
| Safety & oversight | review queue + JSON/Markdown audit export |

## 120-second demo script

1. Hook: "AI teams embed everything — much is waste."
2. Create/load **project** → one vertical PDF corpus.
3. Hands-off: agents run once, log streams.
4. **Proof:** "50% tokens cut, **92% understanding** retained; random cut **68%**."
5. Close: "You pick quality floor; we find max cut — Datter AI Hands-Off."

## Defer tonight

- Adisorn / hybrid scorers
- PhD simulation CSV
- Full M2 greedy selector (unless time after proof tab)
- ML meta-papers as demo input
