# Business Strategy

Durable GTM and positioning from the 2026-06-25 home-workspace Cursor chat.

## Core reframe

The bottleneck is not the datacenter. The bottleneck is **money committed before anyone knows if the data is learnable**.

Cost multiplier (from pitch deck):

```text
Storage ($27k) → Labeling ($150k) → GPU/training ($400k+)
```

Datter sits at the **input gate** — before that chain compounds.

> Move from "Is the data clean?" to "Is the data learnable?"

## What is unique

Not dedup. Not generic data cleaning. Not MLOps observability after training fails.

**Category:** data usefulness layer — proactive economic decision before AI spend.

Defensibility stack:

1. ROI projection tied to labeling, GPU, token, storage waste
2. Regime model — useful data between redundant low-complexity and chaotic high-entropy
3. Model-agnostic input gate (LLM, vision, time-series)
4. Research-backed scoring (compression / description-length literature)
5. Proactive vs reactive — act at data understanding, not after training

Pitch line: **Others manage data or models. Datter decides whether data is worth managing at all.**

## Not B2B vs B2C — who holds the budget

There is no real B2C play. Free local audit = product-led B2B.

| Buyer | What they think they're buying | What Datter sells |
|---|---|---|
| AI builder / RAG team | Stop wasting tokens and embeddings | Pre-ingestion audit: keep / drop / compress / review |
| ML lab / LLM startup | Stop labeling and GPU burn on junk | Complexity regime + core-set selection |
| Bank / regulated enterprise | Prove training data quality | Audit trail + ROI + local/VPC deployment |
| Data platform team | Clean up the lake | Same engine; storage + pipeline waste hook |
| Datacenter / cloud | Help customers spend smarter | **Channel**, not primary ICP |

Same product. Different economic story at the door.

## GTM sequence

### Phase 1 — now (hackathon wedge)

- **Buyer:** AI engineer / ML lead about to ingest docs into RAG or fine-tuning
- **Pain:** token waste in RAG, embeddings, fine-tuning
- **Proof:** "1.2M tokens → safely remove 420k before embedding" + quality did not drop
- **Pricing path:** free local audit → $/GB or $/M tokens analysed → team licence

### Phase 2 — expand pipeline stage

Same buyer, new ROI line items: RAG tokens, fine-tuning labeling/GPU, data lake storage.

### Phase 3 — enterprise / regulated

After VPC, audit exports, Airflow/S3 integration. Banks buy governance and defensibility, not "save tokens."

### Phase 4 — platform / channel

Marketplace add-on, GPU cloud partnership, continuous bucket evaluation. Datacenter is Phase 2–3 channel, not founding story.

## First buyer decision (committed for MVP)

**AI builder / ML engineer about to ingest documents into RAG or fine-tuning.**

Everyone else is a second conversation with the same demo reframed.

## Datacenter angle — defer

Hyperscalers often profit from more consumption. Neocloud/GPU hosts may align better later.

Do not lead with "sell to datacenter to optimize data."

## Investable proof bar

Deck claims 30–50% labeling/training waste reduction. To make credible:

1. Run on 3 real messy corpora
2. Show removal/downgrade without retrieval or eval regression
3. Own one vertical story (legal RAG, support KB, internal wiki)
