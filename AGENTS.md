# AGENTS.md - Datter Project Working Agreement

## Purpose

This repo is the source of truth for the Datter AI project: product direction, local app implementation, experiments, and Datter-specific research notes.

Start Datter work here, not in the everyday-life brain.

## Read First

If this is a **new Cursor chat** continuing from elsewhere, start with `CHAT_START.md`.

Otherwise:

1. `brain/00_System/Datter Brain Manager.md`
2. `brain/00_System/Project Map.md`
3. `brain/00_System/Source Map.md`
4. The smallest target note or code file routed by those maps

## Routing Boundary

- Datter product, data-usefulness research, compression/deduplication notes, experiments, app implementation, and demo evidence live in this repo.
- The everyday-life brain should keep only high-level life/project links back to this repo.
- PhD beam-surrogate or structural-dynamics work belongs in the PhD brain, not here.
- Private personal planning belongs in the everyday-life brain, not here.

## Work Rules

- Prefer maps before folders.
- Do not bulk-read paper notes, generated reports, demo data, or PDFs.
- Put reusable research notes under `brain/01_Research/`.
- Put product/source context under `brain/02_Product/`.
- Put experiment plans and result summaries under `brain/03_Experiments/`.
- Keep implementation facts close to code, tests, and README.
- Run `python3 brain/scripts/check_brain.py` after meaningful brain edits.
